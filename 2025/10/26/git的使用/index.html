Git 核心使用指南

Git 是一款分布式版本控制系统，核心作用是追踪文件变更、协作开发和回溯历史版本，避免代码丢失或混乱。

安装 Git 后，需设置全局用户信息（关联提交记录）：

二、核心操作流程（本地仓库）

1. 初始化仓库：在项目文件夹执行 git init ，生成隐藏的  .git  目录（仓库核心文件）。

2. 追踪文件：

  • git add 文件名 ：追踪单个文件

  • git add .  ：追踪当前文件夹所有变更（新增、修改）

3. 提交记录： git commit -m “提交说明” ，将追踪的变更保存为历史版本（说明需清晰，如“修复登录按钮bug”）

4. 查看状态： git status ，随时查看文件是否已追踪、是否有未提交变更。

5. 查看历史： git log ，查看所有提交记录（包含作者、时间、提交说明）。

三、协作与远程仓库（常用 GitHub/GitLab）

1. 关联远程仓库： git remote add origin 远程仓库地址 （如 GitHub 项目的 HTTPS/SSH 链接）。

2. 推送代码： git push -u origin 分支名 （首次推送加  -u ，后续可简化为  git push ）。

3. 拉取代码：

  • git pull origin 分支名 ：拉取远程分支的最新代码到本地（协作前必做，避免冲突）。

  • git clone 远程仓库地址 ：首次下载整个远程仓库到本地。

四、关键场景处理

版本回溯： git reset --hard 提交ID （通过  git log  获取目标版本的ID，谨慎使用，会覆盖本地未提交内容）。​
创建分支： git checkout -b 新分支名 （如  git checkout -b feature/login ，用于开发新功能，不影响主分支）。

**解决冲突：**拉取或合并代码时出现冲突，打开冲突文件，手动修改 <<<<<<< HEAD  到  >>>>>>> 分支名  之间的内容，再重新  add  和  commit 。
