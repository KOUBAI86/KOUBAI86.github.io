Hexo+Git搭建博客

一、准备工具：Node.js + Git + GitHub仓库（基础支撑）

1. 安装Node.js（Hexo依赖）和Git；

​官网下载即可

2. 注册GitHub账号，创建一个名为 你的用户名.github.io 的仓库（仓库名必须严格一致）。

安装Node.js的意义

  • Hexo是基于Node.js开发的工具，依赖Node.js的“包管理系统（npm）”来安装、运行。没有Node.js，就无法执行 npm install 等命令，相当于“没有发动机，汽车无法启动”。

安装Git的意义

  • Git是“版本控制工具”，后续部署博客时，需要用Git把本地生成的博客文件（HTML/CSS等）推送到GitHub仓库，相当于“把本地文件传到网上的传输工具”。

创建GitHub仓库（ 你的用户名.github.io ）的意义

  • GitHub提供“GitHub Pages”服务：只要仓库名严格是 用户名.github.io ，GitHub会自动把仓库里的静态文件（HTML/CSS）解析成可访问的网页，相当于“给你的博客提供免费的线上服务器”。

二、本地搭建Hexo博客

1. 安装hexo

打开git，执行：
npm install -g hexo-cli

2. 初始化博客

3. 本地预览

hexo clean && hexo g && hexo s
浏览器访问 http://localhost:4000 ，即可看到默认博客页面。
安装Hexo： npm install -g hexo-cli

  • 代码意义：
    ​
  •  npm ：Node.js自带的“包管理器”，用于安装/卸载网上的工具包；
    ​
  •  install -g ： -g 是“全局安装”，表示把 hexo-cli （Hexo的命令行工具）装在电脑全局，任何目录都能调用 hexo 命令；
    ​
  •  hexo-cli ：Hexo的“命令行工具”，有了它才能执行 hexo init 、 hexo s 等操作。
    ​
  • 步骤意义：相当于“在电脑上装一个‘博客生成器’，后续用它来创建博客工程”。
    初始化博客： hexo init myblog  +  cd myblog

 

hexo init myblog 代码意义：
​

  •  hexo init ：Hexo的“初始化命令”，会自动创建一个名为 myblog 的文件夹，并在里面生成博客的基础结构（如 source （存文章）、 themes （存主题）、 _config.yml （配置文件）等），相当于“自动搭建好博客的‘毛坯房框架’”。

​cd myblog 代码意义：
​

  •  cd 是“切换目录”命令，进入 myblog 文件夹后，才能执行后续的 npm install 、 hexo s 等命令（因为这些命令需要在博客工程目录内运行）。
    ​

  • 步骤意义：创建博客的“本地工程文件夹”，并进入工程目录准备后续操作。
    安装依赖： npm install
    ​

  • Hexo初始化后， myblog 文件夹里有个 package.json 文件，里面记录了博客运行需要的“依赖包”（如处理Markdown的包、生成HTML的包）；
    ​

  •  npm install 会自动读取 package.json ，下载并安装所有依赖包，相当于“给毛坯房装水电、门窗，让房子能正常使用”。
    ​

  • 步骤意义：补全博客运行需要的“基础功能模块”，没有这一步， hexo s （启动本地服务）会报错。
    本地预览： hexo clean && hexo g && hexo s 

  • 这是3个命令的“合并写法”（ && 表示“前一个命令成功后，执行下一个”），分开解释更清晰：

hexo clean 

  • 代码意义：清理Hexo之前生成的“临时文件”（如 public 文件夹里的旧HTML），避免旧文件干扰新内容的预览。
    ​
  • 步骤意义：相当于“预览前先清空缓存，确保看到的是最新的博客效果”。

hexo g （ g 是 generate 生成的缩写）

  • 代码意义：
    ​
  • 读取 source/_posts 里的Markdown文章，结合 themes 里的主题样式，自动生成对应的静态文件（HTML/CSS/JS），并放在 public 文件夹里；
    ​
  • 比如你用Markdown写的 # 我的第一篇文章 ，会被转换成h1的HTML标签，且带上主题的样式。
    ​
  • 步骤意义：把“易写的Markdown”转换成“能在浏览器显示的HTML”，相当于“把你写的文字稿，排版成能看的网页”。

hexo s （ s 是 server 的缩写）

  • 代码意义：在本地启动一个“临时服务器”，默认端口是4000，浏览器访问 http://localhost:4000 就能看到 public 文件夹里的网页。
    ​
  • 步骤意义：相当于“在本地开一个‘预览窗口’，不用上传到网上，就能先看博客的效果是否符合预期”。

三、关联GitHub并部署(把本地博客传到网上，让别人能访问)

1. 修改hexo配置

打开博客目录下的 _config.yml ，找到 deploy 项，修改为：

2. 安装部署插件

npm install hexo-deployer-git --save

3. 部署到GitHub Pages

hexo clean && hexo g && hexo d

  • 代码意义：
    ​

  • Hexo默认没有“用Git部署”的功能，需要安装专门的插件 hexo-deployer-git ；
    ​

  •  –save ：表示把这个插件添加到 package.json 的依赖列表里，后续重新安装依赖时会自动包含它。
    ​

  • 步骤意义：给Hexo装一个“Git传输插件”，让它能调用Git把文件推送到GitHub。
    部署到GitHub： hexo clean && hexo g && hexo d 

  • 前两个命令（ hexo clean 、 hexo g ）和“本地预览”一致，重点看第三个命令 hexo d （ d 是 deploy 的缩写）：
    ​

  •  hexo d 代码意义：读取 _config.yml 里的 deploy 配置，用Git把 public 文件夹里的所有静态文件（HTML/CSS/JS）推送到指定的GitHub仓库分支。
    ​

  • 步骤意义：相当于“把本地预览好的博客文件，正式传到GitHub的‘服务器’上，别人通过 https://你的用户名.github.io 就能访问”。

四、后续操作（日常维护博客）

1. 写文章：在 source/_posts 新建Markdown文件

  • 意义： source/_posts 是Hexo约定的“文章存放目录”，只要把Markdown文章放这里，执行 hexo g 时就会自动生成对应的HTML页面，不用手动写HTML，相当于“把文章稿放进‘投稿箱’，系统自动排版发布”。

2. 换主题：下载主题到 themes 目录 + 修改 _config.yml 的 theme 项

  • 意义： themes 是“主题存放目录”，修改 theme: 主题名 后， hexo g 会用新主题的样式生成页面，相当于“给博客换一套‘皮肤’，不用重新写所有样式代码”。

3. 更新博客：重新执行 hexo clean && hexo g && hexo d 

  • 意义：每次修改文章/主题后，需要重新清理旧文件、生成新HTML、推送到GitHub，相当于“把更新后的内容重新传到网上，确保别人看到的是最新版本”。

配置Git全局身份

重新执行部署命令
hexo clean && hexo g && hexo d

2. 创建博客文章文本格式改为.md

Windows系统：打开文件所在文件夹，点击“查看”选项卡，勾选“文件扩展名”，使文件后缀名可见。选择要修改的.txt文件，右键点击文件，选择“重命名”，直接将文件后缀名从.txt改为.md，然后按下Enter键确认修改。

3. 怎么查看博客

查看博客分为本地预览和线上访问两种场景，对应操作如下：

一、本地预览（电脑上看效果，不用联网）

在博客根目录的终端执行：
hexo clean && hexo g && hexo s
执行成功后，终端会提示“Server is running at http://localhost:4000”，此时打开浏览器，输入 http://localhost:4000 即可查看本地博客的效果。

二、线上访问（部署后，别人也能通过网络访问）

当你执行 hexo d （部署）成功后，打开浏览器，输入你的GitHub Pages地址（格式为 https://你的GitHub用户名.github.io ），例如你的地址是 https://KOUBAI86.github.io ，直接在浏览器输入该网址即可访问线上博客。
如果线上访问失败，可检查：

1. 部署是否成功（终端无报错）；
2. GitHub仓库名是否为 用户名.github.io （必须严格一致）；
3. 浏览器缓存问题（按 Ctrl+F5 强制刷新）。
